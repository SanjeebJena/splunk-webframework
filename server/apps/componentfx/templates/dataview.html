Data Template View

The Data Template View provides developers with the ability to render data using the client-side templating engine provided by underscore.js. Explaining underscore templates is beyond the scope of this example, but the following examples should help you get started.

Dataview, Django Tag, Embedded Template

In this example, an Underscore template is included as text in the Dataview tag. In this example, the search manager bound to the Dataview uses the rangemap search command to create a range field in the search results that indicates a severity of values.

{% dataview id="single-search-example" managerid="example-single-search" template='
Event Count: <%= results[0].count %>. Status: <%= results[0].range %>
' %} 
{% block content %}
{% dataview id="single-search-example"
                 managerid="example-single-search" 
                 template='<div class="example-1">Event Count: <%= results[0].count %>.  Status: <%= results[0].range %></div>' 
%}
{% endblock content %} 

{% block managers %} 
{% searchmanager id="example-single-search" 
      search="index=_internal | stats count | rangemap field=count low=0-3 elevated=3-6 severe=6-10 default=severe" 
       earliest_time="-15m" latest_time="now" 
%}
{% endblock managers %}

Dataview, Django Tag, External Template

In this example, an Underscore template is included by name in the Dataview tag. The template is embedded in the HTML page itself. In this example, the search manager bound to the Dataview uses the music data to list number of searches per artist.

The template is embedded using a <script> tag. The type attribute is required to prevent browsers from interpreting the script contents as Javascript.

{% dataview id="table-top-artist-searches" managerid="search-top-artist-searches" templateName="top-artist-search-template" %} 
{% block content %}            
<script type="text/x-underscore-tmpl" id="top-artist-search-template" style="display:none">
<table>
  <tbody>
  <% for(var i=0, l=results.length; i<l; i++) { var line=results[i]; %>
    <tr><td><%= line.search_terms %></td><td><%= line.count %></td></tr>
  <% } %>
  </tbody>
</table>
</script>

{% dataview id="table-top-artist-searches"
   managerid="search-top-artist-searches"
   templateName="top-artist-search-template" %}
{% endblock %}            

{% block managers %}            
{% searchmanager id="search-top-artist-searches" 
     search='| inputlookup musicdata.csv | search bc_uri=/browse/search/* | top search_terms | fields - percent | rex field=search_terms mode=sed "s/\+/ /g"'
     cache=True
%}
{% endblock %}

Dataview, Javascript Example

In this example, we invoke the DataView using Javascript, and associate the content of the template with the view programmatically. This technique allows users to invoke external templates using advanced loaders such as require.js 

{% block content %}            
<script type="text/x-underscore-tmpl" id="top-artist-search-template" style="display:none">
<table>
  <tbody>
  <% for(var i=0, l=results.length; i<l; i++) { var line=results[i]; %>
    <tr><td><%= line.track_name %></td><td><%= line.count %></td></tr>
  <% } %>
  </tbody>
</table>
</script>

{% endblock %}            

{% block js %}            
<script type="application/javascript">
    var deps = [
        "splunkjs/mvc",
        "splunkjs/mvc/searchmanager",
        "splunkjs/mvc/dataview"
    ];
    require(deps, function(mvc, SearchManager, DataView) {
        new SearchManager({
            id: "search-top-song-downloads",
            search: "| inputlookup musicdata.csv | search bc_uri=/sync/addtolibrary/* | stats count by track_name | sort count desc | table track_name count",
            cache: true
        });

        new DataView({
            id: "table-top-song-downloads",
            managerid: "search-top-song-downloads",
            template: $('#search-top-song-template').html(),
            el: $('#search-top-song-table')
        }).render();
    });
</script>
{% endblock %}
