Search Control Views

Search bar

The SearchBar view represents a search input that can be bound to a search manager. However, unlike most other views, SearchBar provides a query to the search manager and modifies it rather than taking results from the search manager.

Use the {% searchbar %} tag with the managerid property to specify the search manager.

Options

You can set the following property:

    timepicker: When true,, the search bar includes a time picker.

Example

The following example shows a SearchBar view with a TimePicker and a ResultTable view, all bound to the same search manager. A default search is used, but if a new one is entered, the result table is updated.

{% searchbar id="example-search-bar" managerid="example-bar-search" %} {% resulttable id="bar-table" managerid="example-bar-search" pageSize="5" %}

var SearchBarView = require("splunkjs/mvc/searchbarview");
new SearchBarView({
    id: "example-search-bar",
    managerid: "example-bar-search",
    el: $("#divToHangOn")
}).render();

var ResultTableView = require("splunkjs/mvc/tableview");
new ResultTableView({
    id: "example-resulttable",
    managerid: "example-bar-search",
    pageSize: "5",
    el: $("#divToHangOn2") // seperate div
}).render();

var SearchManager = require("splunkjs/mvc/searchmanager");
new SearchManager({
    id: "example-bar-search",
    search: "index=_internal | head 100 | timechart count by sourcetype span=100s",
});    

// Hooking up events (both JavaScript and Django)
var manager = splunkjs.mvc.Components.getInstance("example-bar-search");
var searchbar = splunkjs.mvc.Components.getInstance("example-search-bar");
var timepicker = searchbar.timepicker;

searchbar.on("change", function() {
    manager.query.set("search", searchbar.val()); 
});

timepicker.on("change", function() {
    manager.search.set(timepicker.val()); 
});

Timeline

The Timeline view displays an event timeline of a given search and allows interactive modification of the search time range. This view modifies and is modified by its search manager.

Use the {% timeline %} tag with the managerid property to specify the search manager.

Options

You can set the following property using templates tags or JavaScript:

    data: The name of the results model.

Example

In this example, TimePicker, Timeline, and ResultTable views are bound to the same search. The time range can be changed by the time picker or the timeline. Custom time ranges can be interactively selected using the timeline.

{% timepicker id="timeline-timepicker" preset="today" managerid="example-timeline-search" %} {% timeline id="example-timeline" managerid="example-timeline-search" %} {% resulttable id="timeline-table" managerid="example-timeline-search" pageSize="5" %}

new TimePickerView({
    id: "example-timepicker",

    preset: "today",

var TimelineView = require("splunkjs/mvc/timelineview");
new TimelineView({
    id: "example-timeline",

    manager.search.set(timeline.val());

Time picker

A TimePicker view can be used by itself to modify the time range for a search manager. This view can be used with a SearchBar view to modify the time range for the search bar's search.

Use the {% timepicker %} tag with the managerid property to specify the search manager.

Options

You can set the following property using templates tags or JavaScript:

    preset: When set, the time picker starts with the selected time range. The possible values are the time labels defined in times.conf (e.g. "Last 24 hours").

Example

In this example, a TimePicker view is bound to the same search as a single value. As the time range changes with the time picker, the value changes based on the new time range.

{% timepicker id="example-timepicker" preset="Last 24 hours" managerid="timepicker-search" %} {% single id="timepicker-single" managerid="timepicker-search" afterLabel="events" %}

    {% searchmanager id="timepicker-search" search="index=_internal | stats count" %}

var TimePickerView = require("splunkjs/mvc/timepickerview");
new TimePickerView({
    id: "example-timepicker",
    managerid: "timepicker-search",
    preset: "today",
    el: $("#divToHangOn")
}).render();

var SingleView = require("splunkjs/mvc/singleview");
new SingleView({
    id: "example-single",
    managerid: "timepicker-search",
    afterLabel: "events",
    el: $("#divToHangOn2")
}).render();

var SearchManager = require("splunkjs/mvc/searchmanager");

    id: "timepicker-search",
    search: "index=_internal | stats count",

    status_buckets: 300
});
