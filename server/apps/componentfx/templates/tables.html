Table Views

Result table

The ResultTable view is a fully-featured Splunk-style results table that supports pagination. Given a search manager, the ResultTable displays the processed results from that search.

Use the {% resulttable %} tag with the managerid property to specify the search manager.

{% resulttable id="example-table" managerid="example-search" pageSize="5" %}
  • Django
  • JavaScript 
{% resulttable id="example-table" managerid="example-search" pageSize="5" %}
var ResultTableView = require("splunkjs/mvc/tableview");
new ResultTableView({
    id: "example-resulttable",
    managerid: "example-search",
    el: $("#divToHangOn")
}).render();

Options

You can set the following properties using templates tags or JavaScript:

    managerid: The search manager for the table.

    data: The name of the results model to use.

    pageSize: The number of results per page.

    fields: The fields to include in the table. In JavaScript this takes a list of strings. In Django it is a space-separated list.

  • Django
  • JavaScript 
{% resulttable id="example-table" managerid="example-search" fields="_time splunkd" %}
var table= splunkjs.mvc.Components.getInstance("example-table");
table.settings.set("fields", ["_time", "splunkd"]);

Events

The result table fires the following events:

    clicked:cell: Fired when a cell is clicked.

    clicked:row: Fired when a row is clicked.

Event table

The EventsViewer view is a fully-featured Splunk-style events viewer that supports pagination and variable formatting. Given a search manager, the EventsViewer displays the events corresponding to that search.

Use the {% eventsviewer %} tag with the managerid property to specify the search manager.

{% eventsviewer id="example-eventsviewer" managerid="example-event-search"%}
  • Django
  • JavaScript 
{% block content %}
